Otro protocolo utilizado por los módems para establecer comunicación se llama handshake protocol. Se utiliza una serie de señales para establecer la comunicación por módem. La siguiente es una descripción de la secuencia del protocolo de handshake, incluido el procedimiento y las señales de datos utilizados para establecer y mantener la comunicación. Tenga en cuenta que algunas de las señales, como Carrier Detect, Request to Send, and Clear to Send, las utiliza únicamente el módem y no pasan por la propia interfaz de comunicación serie. 1. Cargue y ejecute el programa de comunicación. El módem emite la señal TR (Terminal Ready). 2. Marque el teléfono para llamar al otro módem y sistema informático. El módem cambia al modo de datos y emite la señal DSR (Data Set Ready). 3. El otro sistema descuelga y responde. Se emite el CD (Carrier Detect) y la luz correspondiente se ilumina en el panel del módem. 4. Presione una tecla para comenzar a enviar un código de carácter al otro sistema. La señal RTS (Request to Send o Solicitud de envío) se envía desde la computadora al módem y se ilumina la luz correspondiente. 5. El módem transmite una señal a la computadora para indicar que está preparada para aceptar datos para la transmisión. Se recibe la señal CTS (Clear To Send o Listo para enviar). 6. Se envían los datos. La luz SD (SendData) está iluminada en el módem.

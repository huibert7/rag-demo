Los valores ASCII también representan funciones de control como SHIFT, CTRL, movimiento del cursor, caracteres especiales y símbolos. Los sistemas Apple II reconocen 128 valores de códigos ASCII de este tipo. Directamente relacionado con el código ASCII está el valor de la cadena de caracteres o código `CHR$`. El valor del código `CHR$` se usa a menudo en programas BASIC para acceder a un símbolo o función en particular a la que no se puede acceder directamente desde el teclado. Por ejemplo, en Apple II, el comando BASIC `PRINT CHR$(97)` muestra la "a" minúscula. Si examinara los listados de códigos del Programa de diagnóstico del sistema, vería varios casos en los que se invoca un valor `CHR$`. Debido a que los valores CHR$ y ASCII están relacionados entre sí uno a uno, CHR$(97) y ASCII 97 representan la "a" minúscula. La diferencia entre los usos de los dos esquemas de valores de código es que CHR$ se usa para invocar letras y funciones dentro de un programa BASIC, mientras que ASCII se usa para procesar la entrada del teclado en la CPU. El valor ASCII también se usa para convertir (usando la instrucción BASIC ASC) números de cadenas numéricas en valores numéricos que se pueden usar como tales.
